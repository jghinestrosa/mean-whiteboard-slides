MongoDB - Características

  1. Índices. Soporta indices secundarios, permitiendo consultas rápidas y capacidades de indexación únicas.
  2. JavaScript. Los desarrolladores pueden seguir programando en JavaScript en el lado del servidor para interaccionar con la base de datos.
  3. Colecciones de tamaño fijo. Útiles para ciertos tipos de datos a almacenar, como archivos de logs.
  4. Almacenamiento de archivos. Proporciona un protocolo para almacenar archivos de gran tamaño y metadatos.
  • Características propias del modelo relacional no están presentes (joins, transacciones complejas...)
  • Esto se debe a decisiones de la arquitectura para permitir la escalabilidad. Son características dificiles de proporcionar eficientemente en un sistema distribuido.
